Moduliai

Moduliai yra atskiri kodo fragmentai išskaidyti skirtinguose failuose.

Tuose moduliuose gali būti įvairus kodas. Nuo duomenų (kintamųjų) iki funkcijų, klasių, metodų ir kt.
JS klasės turi būti atskiri moduliai. Kitaip pasakius - viename faile negali/neturėtų būti aprašytos kelios klasės.
HTML'e prijungtas script failas turi turėti nurodytą modulio tipą.
HTML failas, kuriame yra modulio tipo JS failas, privalo būti paleistas kaip serveris, nes kitu atveju neleis vykdyti JS failo kodo.

Export ir Import

Failuose esanti informacijas gali būti iškeliama (export'uojama) ir galima kažkokią informaciją įsikelti iš kito failo (import'uoti).

Export

Informaciją galima eksportuoti dvejais būdais.
1) export default - numatytasis/pagrindinis export'as gali būti TIK VIENAS viename faile.
2) export - papildomas export'as/ai, kurių gali būti daug viename faile.

Import

Informacija galima importuoti įvairiais būdais.
1) import sameNameAsExportDefault from ./someFile - gaunama default export informacija. 2) import differentNameAsExportDefault from ./someFile - gaunama default export informacija. 3) import { sameNameAsExport, sameNameAsExport2, ... } from ./someFile - gaunama nurodyto pavadinimo export informacija (ne export default, o paprasto export). 4) import { sameNameAsExport as norimasNaujasPavadinimas, sameNameAsExport2, ... } from ./someFile - gaunama nurodyto pavadinimo export informacija, kurią šiame faile vadinsi kitu susigalvotu pavadinimu (ne export default, o paprasto export). 5) import pagrindinisImport, { nePgrImport, nePgrImport2 as kitasPavadinimas } from ./someFile - galima vienu import'u importuoti ir export default ir tiesiog export.
